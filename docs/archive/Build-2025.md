# Microsoft Build 2025 Deep Research

### Microsoft 365 CopilotチューニングとTeams向けエージェント開発強化

* **Copilot Tuning:** **Microsoft 365 Copilot Tuning** が発表され、企業が自社ドメインの知識で生成AIモデルを迅速かつ高精度に調整できるようになります。例えば法律業界では、過去の文書スタイルや専門知識を反映した契約書草案を生成するAIエージェントを構築可能です。また**Agent Builder**で構築した社内エージェントから、この調整済みモデルを利用できます。Copilot Tuningは**5000ライセンス以上**のMicrosoft 365 Copilot契約ユーザー向けに6月からプライベートプレビュー提供開始予定です。
* **Teamsコラボレーションエージェント開発:** Microsoft Teams上で動作するエージェント開発用に、先進のツール類がプレビュー公開されます。**Agent2Agent (A2A) プロトコル**対応により、複数エージェント間でP2Pのセキュア通信が可能となり、仲介サーバーなしでデータや認証情報をやりとりできます。また**Teams AIライブラリ**のアップデート（JavaScript/C#向け）により、最新AI機能を使ったカスタムエージェント開発が容易になりました。**エージェントメモリ**機能によりエージェントがユーザーとの対話履歴を記憶し、より文脈に沿った個別対応が可能です。さらに**エージェント検証自動化**により、ストアポリシー遵守や品質チェックを自動化して開発効率を高めます。**導入・エンゲージメント分析**機能では、開発者ポータルでリアルタイム利用状況やエージェントの採用率などの分析指標を提供し、継続的な改善に役立てられます。加えて、**Teams会議用エージェント**も実現し、会議画面から直接エージェントを呼び出したり、新しい **Insights API** で会議のAI生成ノートにアクセスするエージェントも構築可能になります。これらの機能群は2025年5月より順次プレビュー提供されます。

### Microsoft Copilot Studioの新機能拡充

* **プロコード向け開発ツール:** **Microsoft Copilot Studio** と **Microsoft 365 Copilot** に、プロ開発者向けの新機能が多数追加されました。まず**Microsoft 365 Agents Toolkit for Visual Studio**が一般提供（GA）となり、Visual Studio上でのエージェント開発を効率化します。これにより、Microsoft 365 Agents SDKやAzure AI Foundry等のAIツールとプロジェクトのひな形作成・テスト・デプロイが統合され、さらに型定義言語「TypeSpec for Copilot」も統合されました。また**Microsoft 365 Agents SDK**自体もGAになり、大規模でマルチチャネルなエージェント構築を支援します。Azure AI Foundryによるモデルカスタマイズや、Copilot Studio・Visual Studioとのシームレス統合、10以上のチャネルへの展開機能を備えています。さらに**Microsoft 365 Copilot API**（エンタープライズ対応のAPI群）が公開され、Microsoft 365のデータや検索・チャット機能を外部アプリから利用可能にします（検索APIは現在プレビュー）。
* **エージェントエコシステムと接続:** **Agent Store**（エージェントストア）の一般提供により、Microsoftやパートナー、顧客組織が公開したエージェントを発見・共有・利用できる統合カタログが利用可能になります。Microsoft 365 Copilotの各エンドポイント（Copilot Chatなど）でパーソナライズされたエージェント推奨や検索ができ、ユーザーは自分に必要なエージェントを素早く見つけ共同作業に活用できます。また**Power Platform コネクタ SDK**が強化され（プレビュー）、開発者はエンタープライズのデータソースとAIエージェントを繋ぐコネクタを迅速に作成可能です。新SDKで構築したコネクタはエージェントが企業内の構造化データへ容易にアクセス・推論できる形式を提供します。さらにAzure AI Foundryの**独自モデル持ち込み (BYOM)** 機能で、Copilot Studio製エージェントからFoundry提供の**1,900以上**のAIモデルにアクセスでき、業界固有の精細なモデル（特定シナリオ向けプロンプトや要約モデル等）も利用可能になります（プレビュー）。加えて、**Dynamics 365のデータ連携**も進展し、Microsoft 365 CopilotがDynamics 365の営業・サービス・サプライチェーン・マーケティングデータにアクセスして洞察を得られるようになります（現在プライベートプレビュー）。
* **ローコード向けエージェント開発強化:** Copilot Studioはローコード開発者向けにもツール群を拡充しています。**マルチエージェントシステムのサポート**（プレビュー）により、一つのソリューション内で複数のエージェントが**チーム連携**できるようになります。Copilot StudioやAzure AI Foundry Agent Service、Microsoft 365 Agents SDKで構築されたエージェント同士がオープン標準（A2Aプロトコル等）で通信し、専門スキルを組み合わせてより包括的な回答や処理を実行できます。また**Model Context Protocol (MCP)** のCopilot Studio対応がGAとなり、エージェントが外部システムを横断して統一的にデータ・モデルへアクセス可能になりました。特にDataverseおよびDynamics 365用のMCPサーバー（プライベートプレビュー）により、エージェントがこれら業務データに安全に接続し操作できます。
* **エージェントの外部ツール利用:** **Computer Using Agent (CUA)** と呼ばれる機能がFrontierプログラム経由で提供開始され、エージェントがPC上のGUIアプリ（デスクトップやWeb）を操作して業務タスクを自動化できるようになります。例えば、人手で行っていたデータ転記や文書処理、市場リサーチ、コンプライアンス監視などを、エージェントが実際に画面上でクリックや入力を行う形で代行可能です。大規模実行にも対応し、自社またはMicrosoftホストのVM上でエージェントを動かせます（現在、米国環境でCopilot Studioメッセージ50万件以上の一部顧客に限定提供）。さらに**ツール拡張**として、既存のツール/アクション（ドキュメント処理、深い推論プロンプト、エージェントフロー、各種コネクタ、REST API等）全般の強化が図られ、エージェントがより高度な操作を行えるようになりました（プレビュー）。**コードインタープリタ**機能も追加され、エージェントがPythonコードを自動生成・実行して計算やデータ分析・可視化など複雑な処理をこなせます（プレビュー）。また**Dataverse for Teamsエージェント用データベース**がプレビュー提供され、エージェント向けの高速リアルタイムDBを利用可能に、**Dataverse検索**機能（GA）により組織内外の構造・非構造データを統合検索し知識ネットワーク化することもできます。これらによりMicrosoftは、Copilot Studioを中心とした包括的なエージェント開発プラットフォームを提供し、開発者や企業が自社業務にインテリジェントなAIエージェントを容易に構築・管理・展開できるようにしています。

### Microsoft 365 Copilot新機能 (Wave 2リリース)

* **Wave 2春リリース:** Microsoft 365 Copilotの「Wave 2」アップデートがリリースされ、多数の新機能が追加されました。まず**Microsoft 365 Copilotアプリ**自体がアップデートされ、人間とAIエージェントのコラボレーションを一層深める設計になっています。また、**OpenAI GPT-4ベースの画像生成**機能をCopilotに組み込んだ新しい「生成 (クリエイト) エクスペリエンス」が提供され、テキストから画像コンテンツを生成可能になりました。さらに、データやコンテンツを即座に洞察・アクションに変換する**Copilotノートブック**が一般提供開始されました。将来的な機能として、**Copilotサーチ**（業務用検索AI）と**Copilotメモリー**（長期記憶機能）が2025年6月より順次展開予定です。加えて、**Researcher**と**Analyst**という役割別AIエージェント（業務リサーチ用の初の推論AIエージェント）が登場し、最先端ユーザー向けプログラム（フロンティアプログラム）を通じて今月から一部顧客に提供が開始されています。
* **Outlook向けCopilot拡張:** Outlookのメール・予定表に対するCopilot機能が正式リリースされ、日々の情報整理と業務準備を効率化します。具体的には、**メール検索結果の要約**や**添付ファイルの内容プレビュー**をメール一覧上に直接表示し、膨大な受信トレイを効率よくナビゲートできるようになります。また、会議予定に関連する**トピックの文脈情報やTo-Do、関連ドキュメント**等をCopilotが自動収集・要約し、ユーザーが会議前に素早く準備できるよう支援します。これらにより、メール処理や会議準備にかかる時間を削減し、重要な判断・対策に集中できるようになります。
* **Copilot Pagesの新ワークフロー:** **Microsoft Copilot Pages**（Copilotによる動的なドキュメント生成・共有機能）に新機能が追加され、生産性向上につながります。まず**モバイル端末からのワンクリックページ作成**機能がGAとなり、スマホ上のCopilot Chatから即座に新しいページを生成し、その場で共有・編集したり、後でPCのWebから続きを編集するといったシームレスな利用が可能です。次に**ワンクリックでPagesをWord文書化**する機能が追加されました。Copilot Pagesは共同編集に適したフォーマットですが、従来型のWordやPDFで内容を出力・配布したいニーズにも対応し、ボタン一つでページ内容をWordファイルに変換できます。さらに、**Pages一覧の検索・フィルタ機能**がCopilotアプリ内に実装され、蓄積したページを容易に探せるようになりました。また、Copilotチャットの**出力オプション強化**として、**対話型チャート**や**コードブロック**を出力できるようになりました。ユーザーが「このデータをチャートにして」と頼むと、Copilotが理解しやすいグラフに変換してページに追加するといったことが可能です。これら新機能は2025年5月より一般提供が開始されました。

### Azure AI Foundry: 新モデル追加とエージェントサービス一般提供

* **最新AIモデルとパートナー連携:** AzureのAI開発基盤「**Azure AI Foundry**」において、最新の大規模モデル群がカタログに追加されました。Elon Musk氏のxAIによる新モデル **「Grok 3」** は**本日より利用可能**となり、さらにBlack Forest Labs社の **「Flux■Pro 1.1」** や、日本発のモデル **「Sora」** も近日中にAzure OpenAI経由でプレビュー提供予定です。加えて、AIコミュニティの重要なリソースである **Hugging Face** の**1万種以上**のオープンソースモデルがFoundry上で利用可能になりました。開発者は多彩なモデル選択肢から、自身のアプリやサービスに最適なモデルを選び出せます。さらに、**モデルのフル微調整（ファインチューニング）** サポートが追加され、開発者は大規模モデルを自社データに合わせてカスタマイズ可能となりました。この微調整機能のために、新たに\*\*開発者向け層（デベロッパープラン）\*\*も導入され、**ホスティング料金無料**でファインチューニングが試せるようになっています。
* **Azure AI Foundry エージェントサービス (GA):** Azure AI Foundry上で動作する**エージェントサービス**が一般提供となり、エンタープライズグレードのAIエージェントを設計・デプロイ・拡張できるようになりました。このサービスは**マルチエージェントワークフロー**をサポートし、開発者は複数の専門AIエージェントをオーケストレーションして複雑な業務タスクを自動化し、意思決定の迅速化や効率向上を図れます。また**Agent2Agent (A2A)** や **Model Context Protocol (MCP)** などのオープンプロトコルにも対応し、異なるエージェントフレームワーク間の相互運用を実現しつつ、オーケストレーションの精密な制御が可能です。Foundry Agent Serviceは社内の**知識ソース**（例: Microsoft Bing検索、SharePointドキュメント、Azure Databricksデータ、Microsoft Fabricデータウェアハウス等）と容易に接続できるため、エージェント開発を簡素化できます。さらに開発者がすぐ利用できる**エージェントテンプレート集**（プレビュー）も提供され、典型的な業務シナリオ向けの雛形からエージェントを構築可能です。このサービスはSemantic KernelやAutoGenの機能を統合した**共通ランタイム**上で動作し、エージェントのローカル開発→クラウドデプロイを一貫した体験で行えます。開発者はローカル環境でエージェントシステムをシミュレーションし、動作確認後にそのままAzureクラウドにデプロイでき、AIエージェント開発の生産性と信頼性が飛躍的に向上します。

### エージェント型DevOpsとGitHub Copilotの新機能

* **自律型エージェントによるDevOps:** マイクロソフトは**Agentic DevOps**のビジョンを発表し、ソフトウェア開発ライフサイクル全体にAIエージェントを組み込むことで、開発・運用プロセスの自動化と高速化を図ります。具体的には、GitHub Copilotに**開発タスクを自律的に割り当てるAIエージェント機能**が追加されました。開発者はGitHub Copilotに対し、コードのリファクタリング、テストカバレッジ向上、不具合修正、新機能実装などを**タスクとして指示**できます。複雑な案件では、Copilotが他の専門エージェント（テストエージェントやデプロイエージェント等）と協調し、企画からリリースまで対応します。これにより定型コーディング作業はAIに任せ、開発者はより創造的なタスクに集中できるようになります（※Copilotコーディングエージェント機能は現在プレビュー）。
* **レガシーアプリのモダナイズ支援:** **GitHub Copilotのアプリ近代化機能**も強化されました。開発者はCopilotに、古い**Java**や\*\*.NET**アプリのアップグレード作業を指示することで、大量のコード変換や依存関係更新といった煩雑作業を自動化できます。例えば、Azure上で稼働するJavaアプリのクラウド最適化では、Copilotが数千ファイル規模のコード評価から修正提案まで行い、従来数日～数ヶ月かかっていた移行作業を数時間程度まで短縮します。.NETやJavaのバージョン更新も、Copilotが**自動アップグレードプラン\*\*を生成し、各ステップを実行・検証、最終的なサマリーを提示して完了します。これによりアプリのパフォーマンス向上やセキュリティ脆弱性解消、技術的負債削減を効率的に行えるようになります（プレビュー提供中）。
* **SREエージェント:** 信頼性エンジニアリング向けの**SREエージェント**も新登場しました。このエージェントはクラウド上の本番アプリを常時監視し、**アラート対応や自動緩和策**を講じてサービス正常性を維持します。従来、SREはアラート疲れや手動対応の負荷に悩まされていましたが、SREエージェントは発生したアラートに即応し、自律的に問題を切り分け（原因分析）して解決まで実行します。例えばKubernetesやPaaS、サーバーレス、データベースサービス上のアプリを監視し、異常検知次第、**数分以内**に対処を完了するため、復旧時間を従来の数時間から大幅に短縮できます。このエージェントはAzure Monitorなどからの**高度な脅威検出**（15種類以上の検知パターン）に基づいて動作し、GitHub Copilotの他エージェントとも連携して、問題発生時には自動で開発エージェントにIssueを作成・割り当てして修正に導くことも可能です。SREエージェントは**数週間以内にプレビュー**提供開始予定です。
* **GitHub Models:** 開発者が様々なAIモデルを試せる **「GitHub Models」** プラットフォームもプレビュー公開されました。これはAzure AI Foundryの強力なAIモデル群を**GitHubにネイティブ統合**するもので、GitHub上のリポジトリから直接モデルを呼び出し、プロンプトの実験・評価が行えます。Pull Requestやコミット、CI/CDといった**GitHubの標準ワークフロー**にAIモデル活用を組み込み、開発サイクル内でモデルの比較検討やプロトタイピングを迅速に行うことができます。OpenAIやMeta、Cohere、Microsoft、Mistralといった主要提供元の最新モデルにすぐアクセスし、コストや精度を比較しながら最適なモデルを選定可能です。モデル評価プロセスを自動化することで、開発者はわずかな時間でAIアプリやエージェントの試作から本番デプロイまで到達できます（GitHub Modelsは現在プレビュー）。
* **Copilotエージェントモード:** **GitHub Copilotのエージェントモード** (Agent Mode) 機能がVisual Studio CodeおよびVisual Studioでプレビュー開始されました。エージェントモードでは、Copilotが真の**ペアプログラマー**のように振る舞い、自動補完以上の高度な支援を行います。具体的には、開発者のコードベース全体を理解し、複数ファイルにまたがる大規模リファクタリングやテスト実行、エラーログ分析まで、一つのチャットプロンプトから**一連のコーディングタスクを自動実行**できます。必要に応じてターミナルコマンドの提案や実行も行い、コードが壊れた場合は**自律的に修復**を試みることも可能です。このモードにより開発者の手間を大幅に軽減し、生産性を高めます。Agent Modeは現在VS Code/VSで利用でき、今後JetBrainsやEclipse、Xcodeといった他のIDEへの拡張も予定されています（プレビュー版）。

### Power PlatformとビジネスアプリへのAI統合

* **Power AppsのAI支援機能:** Power Appsには新たに**AI活用の開発機能**が搭載され、ローコード開発者とエンドユーザー双方にとってエージェントとの協調が深化しました。まず**Solution Workspace**（ソリューション作業スペース）が一般提供され、統合開発キャンバス上で**AIエージェントと協働して要件定義やデータモデル設計**ができるようになりました。このWorkspaceでは、チャットGPTのようなエージェントがビジネス要件に基づきデータテーブルやプロセスマップを自動生成し、アプリ＋フロー＋Copilotエージェント＋ページ＋ダッシュボードを組み合わせた**総合ソリューション設計**を支援します（Copilot Studio製エージェント統合は5月30日にGA予定）。また**自然言語でのUI開発**プレビュー機能により、開発者はPower Apps Studio上で「○○なページを作って」といった指示を与えるだけで、裏側ではReactコード付きの完全なアプリページが自動生成されます。これによりUI構築が劇的に簡素化し、コーディング知識が乏しくても高度なアプリが迅速に構築できます（早期アクセス提供中）。加えて、**「アプリ用エージェントフィード」** 機能が登場しました。各アプリにエージェントの活動状況フィードを埋め込むことで、ユーザーはアプリ内でどのエージェントが何をしているか常に把握でき、必要に応じて人間が介入すべきポイントでは関連画面へのナビゲートが行われます。さらにこのフィードでは、過去データや現在のコンテキストから**エージェントがおすすめアクションを提示**することもでき、ユーザー自身がテンプレ化された自動処理（ワークフロー）を素早く作成する手助けにもなります。もう一つ重要な強化として、**コードファースト開発のPower Platform取り込み**があります。プロ開発者は好きな言語・フレームワークでコーディングしたアプリをPower Platformに持ち込み、Managed環境でホストさせることが可能になりました。例えばVisual Studio Code上で、AIアシスタントに「このデータソースのAPIを教えて」と質問し、提示されたAPIコードスニペットを取り込みつつアプリを実装、そのアプリをPower Platformにデプロイすると、従来のPower Appsで作成されたものと同様に**フローやエージェント**と接続して実行できます。このコードファースト統合は現在早期アクセスプログラムで提供中です。
* **Power Pagesのポータル開発強化:** **Microsoft Power Pages**（外部向け業務ポータル構築サービス）にも多数のアップデートが行われ、プロ開発者にとって一層扱いやすいプラットフォームになっています。まず\*\*「独自コードの持ち込み」\*\* 機能により、サードパーティの次世代コード生成ツール（生成AI）で作成したHTML/JS/CSSなどのコードをPower Pagesにシームレスに組み込めるようになりました。これにより自然言語でコード生成→サイト反映という開発手法が可能となり、プログラマーはAIの生成したコードを**レビュー・テスト・最適化**する役割にシフトします。AI支援により生産性を高めつつ、自社のコーディング規約やセキュリティ標準も保てる柔軟な開発プロセスが実現します（この機能はプレビュー中）。次に**Visual Studio Code統合**（プレビュー）により、VS Codeから直接Power Pagesのサイトをプレビュー表示したり、UI上でCLIコマンドの実行・環境切替ができるようになりました。これによりクラウドポータルを編集する際にも開発者が馴染みのIDEから離れずに作業でき、生産性向上につながります。さらに**多言語対応機能**がGAとなり、45以上の既存言語に加えて任意のカスタム言語でサイトを構築可能になりました。フォーム、リスト、複数ページフォーム、カード表示など全ての標準コンポーネントについて、多言語コンテンツを管理するための「コンテンツスニペット」が提供され、ユーザーは選択した言語ごとにUIテキストを翻訳・配置できます。これにより、多国語ポータルの構築作業が飛躍的に効率化されます。Power Pagesへの**エージェント追加**機能も新設されました。Copilot Studioで作成したエージェントを含め、複数のAIエージェントをWebポータル上に統合し、サイト利用者がそれらエージェントと対話して業務データを扱えるようになります。例えば、ユーザーがページ上のQ\&Aチャットエージェントに問い合わせると、そのエージェントがDataverse上のデータを安全に参照して回答したり、別のタスク実行エージェントに引き継いで処理したりできます。ユーザーは複数エージェント間をシームレスに切り替えて利用できるため、従来のフォーム送信以上に**インタラクティブなUX**を実現できます。このエージェント統合は高度な対話型ポータル実現の鍵となる機能です。その他、**ビジネスルールのクライアントAPI**がGAとなり、Power Pages内でビジネスルール（入力検証や動的表示制御など）をクライアントサイドで簡単に実装できる堅牢な関数群が提供されました。**インテリジェントなリスト検索・カスタマイズ**機能（プレビュー）も追加され、自然言語のクエリで大量データを絞り込んだり、AIによる結果の要約表示などが可能になっています。なお、**Copilot in Power Pages**（生成AIによるページ構築アシスタント）は既にGAとなっており、新しいページのデザインや既存テンプレート活用を対話形式で行えます。これらのアップデートを通じて、開発者はよりダイナミックで応答性の高い安全なビジネスポータルを迅速に構築でき、Power Pagesを核とした業務向けWeb開発がさらに革新されます。
* **Dynamics 365とAIエージェント (MCPサーバー):** 企業の基幹業務アプリであるDynamics 365もAIエージェントとの連携強化に向けた新機能が発表されました。プレビュー提供の**MCPサーバー (Model Context Protocolサーバー)**により、AIエージェントがDynamics 365のデータや機能に安全にアクセスできる**標準プロトコル層**が実装されます。これにより、Copilot StudioなどMicrosoftのエージェント開発プラットフォーム上で構築したエージェントや、サードパーティ製のカスタムエージェントから、Dynamicsアプリの持つ顧客情報や在庫、営業プロセス等に統一的に接続できるようになります。具体的な強化点として、**Dynamics 365各アプリがMCPネイティブ対応**となり、CRMやERPの各機能がエージェント主導の自動化エコシステムに直接参加できるようになります。Copilot Studioはそのハブとして機能し、MCP標準経由で全てのDynamics 365アプリと通信できる**最高のMCPホスト**となります。複数のDynamics製品を運用する企業は、部門やアプリの垣根を越えて**横断的なエージェントワークフロー**を構築可能となり、真のエンドツーエンド自律業務プロセス（例: 受注→製造→出荷→請求までを各システム横断でAIが調停）を実現できます。また、MCPの導入により**エージェント開発自体も簡素化**されます。開発者やパートナーはDynamics用エージェントを作成する際、各アプリ個別のAPIではなく統一プロトコルを利用できるため、複雑さが低減し、ソリューション提供までの時間が短縮されます。これらによりDynamics 365はAI時代の**自律型企業**への移行を強力に後押しし、業務アプリケーション領域でもエージェント協調が進む見込みです。

### Microsoft EdgeにおけるAI機能の拡張

* **ブラウザ組込みAIモデル/API:** Microsoft Edgeブラウザに**新しいAI APIファミリー**が追加され、ウェブ開発者はEdge内蔵のAIモデルを自分のWebアプリに手軽に組み込めるようになりました。クラウドAIはコストやプライバシーの懸念がある一方、オンデバイスAIは導入が複雑でしたが、EdgeのAI APIによりそのギャップを埋めます。EdgeはMicrosoftが独自開発した\*\*「Phi-4-mini」**という約38億パラメータの小型言語モデルを内蔵しており、開発者はページや拡張機能からこのモデルにアクセスできます。提供されるAPIには、任意のプロンプトをモデルに投げるための**Prompt API\*\*、および**Writing Assistance API**（テキスト生成・要約・編集用）のセットが含まれ、これらは**開発者向けトライアル**として既に利用可能です。さらに**Translator API**（テキスト翻訳）が今後数ヶ月以内にリリース予定です。これらのAPIはEdgeブラウザにビルトインされているため、Webアプリ側で複雑なMLモデル管理をする必要がありません。頻繁に使う生成系AIタスクをEdgeにオフロードでき、開発コストやユーザー端末の負荷を抑えることができます。特に医療・金融など機密データを扱うシナリオでは、データをクラウドに送信せず**オンデバイス処理**で完結できるためプライバシーとセキュリティが向上します。これらEdgeの実験的API群は将来的に**Web標準化**も目指されており、他ブラウザや様々なAIモデルでも動作可能な仕様として検討されています。現在Edge Canary版およびDev版で試用でき、フィードバックが開発チームに募集されています。開発者はEdgeが提供する軽量かつ強力なオンデバイスAI機能を活用することで、新たなユーザーエクスペリエンスを創出できるでしょう。
* **PDF文書のワンクリック全頁翻訳:** Microsoft Edgeは新機能として、PDF文書を**丸ごと別言語に翻訳**する機能を追加します。これまで異なる言語のPDFを読むには、必要部分をコピー&ペーストして翻訳ツールにかけるなど手間がかかり、文脈が途切れがちでした。EdgeではPDFを表示してアドレスバーの「翻訳」ボタンをクリックするだけで、選択した言語に**全文翻訳された新しいPDF**が即座に生成されます。70以上の言語に対応し、しかもレイアウトを保ったままリアルタイムに置換するため、ユーザーは元のフォーマットで内容を追いながら理解できます。海外製デバイスのマニュアルや多国籍企業の社内資料なども、数クリックで母国語化できるため生産性が向上します。PDF全頁翻訳機能は**来月GA予定**で、Edge Canary版では一足先に試用可能です。
* **Edge for BusinessのCopilotエージェント:** ビジネス用途向けEdgeブラウザに、Microsoft 365 Copilotの**エージェント機能**がプレビュー導入されます。Copilot Chatのサイドバーから、特定業務に特化したAIエージェントを呼び出し、繰り返し作業の自動化や情報要約を行えるようになります。例えば**営業支援エージェント (Sales Assistant Agent)** を起動すると、リード情報の収集や顧客とのやりとり履歴のまとめ、売上分析の生成などを自動で行ってくれます。ユーザーは閲覧中のWebページから離れることなく、ブラウザのCopilot枠内でこれらエージェントの結果を得られるため、作業フローが中断されません。またEdgeのCopilot Chatには**ドキュメント要約**機能も追加されました。ブラウザで開いているMicrosoft 365のWord/Excel/PowerPointオンライン文書を、サイドバーのCopilotが即座に要約して表示します。長大なレポートや複雑なスプレッドシートもポイントだけ抽出されるので、内容把握にかかる時間を短縮できます。この要約機能は現在Microsoft 365 Copilotライセンスを持つ商用ユーザー向けにプレビュー提供されています。エージェント機能（Sales Assistant等）は2025年6月上旬からEdge for Business内で利用可能になる予定です。

### AIセキュリティとガバナンス: Purview統合とDefender強化

* **Microsoft Purview SDK (プレビュー):** 開発者がAIアプリに企業レベルのデータセキュリティ対策を組み込むための **Microsoft Purview SDK** が発表されました。REST APIとサンプルコードが提供され、開発中のAIアプリから直接Microsoft Purviewの情報ガバナンス機能（機密ラベルやデータ残存ポリシー等）を呼び出せます。例えば、APIを用いてAIへの**プロンプトや生成結果をリアルタイム分類**し、機密データが含まれていれば応答生成をブロックする、といった制御をアプリ内で実装できます。セキュリティ管理者は既存のPurviewポリシーをそのままAIソリューションにも適用できるため、AI活用が進む中でもコンプライアンスを保ちながら開発を進められます。
* **Azure AIとのネイティブ統合:** Microsoft Purviewは今後、Azure AIサービス群と深く統合し、**生成AIの利用リスク**に対する可視性と制御を提供します。プレビュー段階の新機能として、**Purview Data Security Posture Management for AI** が発表されました。これはデータセキュリティ管理者が、Azure AI Foundry上で開発者が構築・デプロイしたカスタムAIアプリ/エージェントについて、ユーザーの入力プロンプトやAI応答内に含まれる機密情報、あるいは不適切な有害コンテンツ（暴力・ヘイトなど）の使用を検知できるようにするものです。さらに管理者は、社内ユーザーごとのAI利用状況をリスクレベル別に分析し、リスク軽減のための**推奨アクション**（特定ユーザーの利用制限等）を受け取ることもできます。また**インサイダーリスク管理**との連携により、例えば退職予定者が社内のAIアプリを使って異常に大量の機密データを取得している場合に自動アラートを上げ、該当ユーザーのアクセス遮断等の対応が取れるようになります。そして**AIとの全てのやり取り内容をPurview監査ログに記録**する機能も実装されます。コンプライアンス管理者はこのログに基づいてデータ保持ポリシーを適用し、一定期間後にAI対話記録を自動削除したり、eディスカバリで調査用に検索・抽出したりできます。さらにコミュニケーションコンプライアンスの仕組みと組み合わせて、AI経由で共有された不適切コンテンツを検知・是正するといったことも可能になります。
* **Defender for Cloud のAI保護:** Microsoftのクラウドネイティブアプリ保護プラットフォーム(**Defender for Cloud**)が、Azure AI Foundry環境と連携し強化されます。Foundryポータル上で稼働するAIサービスの**セキュリティ状況モニタリング**と**ランタイム脅威防御**を統合するもので、開発者とSecOpsが協調してAIワークロードの安全性を確保できます。具体的には、Defender for Cloudから直接、Azure AI Foundry上のプロジェクトに対する**脆弱性情報や設定ミス**の指摘を受け取り、修正のベストプラクティスガイダンスを得られます。またAIサービス固有の**脅威検出アラート**が提供され、例えばプロンプトへの攻撃（プロンプトインジェクション）や不正なAPIキー利用、機密データ漏洩の兆候など、15種類以上の検知シグナルに基づいてリアルタイム警報が出されます。開発者はAzure上で動くAIモデルがどのようなセキュリティリスクに晒されているか可視化でき、検知内容に応じて適切な緩和策（設定変更やキー無効化等）を即座に講じられます。DefenderによるこうしたAIワークロード保護は、AIプロジェクトが増える中で**セキュリティをシフトレフト**し、開発段階から本番運用まで一貫して安全対策を適用するための重要なソリューションとなるでしょう。
